Heaps RPG Demo

Controls

  • Arrow keys to walk
  • Space to interact with the NPC (will cause him to reverse direction so you can corrale him around the map)
  • F1 - Show collision permissions
  • F2 - Show grid
  • F3 - Show current player position

About This Demo

This is a demo of the heaps-rpg library, which is intended as a resource for build 2D topdown RPG games in Heaps.io. The scope of the demo is the abilitiy to walk around an extremely barebones Palette Town from Pokemon Fire Red on the GBA.

Although you can't actually do much in the demo, it does showcase many complex features.

  1. Layer-based map rendering
  2. Walking player
  3. Lockable/Releaseable controls
  4. Grid-based walking
    • Grid-based walking with multiple entities, including tile reservation
  5. Collision permissions
  6. Changing maps
    • On step warping
    • Custom warping workflows (i.e. trying to step on a door causes a chain of commands: lock player -> open door -> walk up -> fade out)
  7. Interacting with what the player is facing

... and a lot of junk under the hood.

Notably absent features currently:

  1. Animated tilesets

    - This can easily be added as a custom layer type, it just seemed like a lot of hassle to configure for the demo

  2. Textboxes

    - Yeah, textboxes are a huge subject... They are quite complex to implement and probably require custom implementations per-project. I have a heaps textbox/dialogue box library which I can share, and with this framework it should be easy to include any textbox no issue.

    - Currently, as far as I'm concerned, this framework is BYOTB (bring your own textbox).
